Шаблоны функций

Обобщенное программирование

Рассмотрим следующую функцию:

unsigned nth_power(unsigned x, unsigned n) {
    unsigned result = 1;
    if (x < 2 || n == 1)
        return x;
    while (n > 0) {
        if ((n & 0x1) == 0x1) { // проверка n на четность 
            result *= x;
            n -= 1;
        } else {
            x *= x;
            n /= 2;
        }
    }
    return result;
}

Текущая функция работает не для всех типов, а только для тех, что являются unsigned, те такие типы как, например, float здесь неуместны. Для того, чтобы исправить эту проблему, мы можем добавить стандартный template к нашей функции:

template<typename T> 
T nth_power(T x, unsigned n) {
    T result = 1;
    if (x < 2 || n == 1)
        return x;
    while (n > 0) {
        if ((n & 0x1) == 0x1) { // проверка n на четность 
            result *= x;
            n -= 1;
        } else {
            x *= x;
            n /= 2;
        }
    }
    return result;
}

Стало намного лучше. Однако, в такой реализации функции у нас встречаются следующие банальные проблемы:

  • Корнер-кейс для x не сработает для типа float
  • Если наш T будет матрицей? В таком случае, T result = 1; будет некорректным

В таком случае возникает банальный вопрос: как исправить нашу функцию так, чтобы эти две проблемы исчезли? На самом деле очень просто: подогнать наш алгоритм под типы. Для этого мы можем воспользоваться одним из двух следующих методов:

  • Добавить специальный default_id_trait в template
  • Вынести главный алгоритм нашей функции в другю функцию и подогнать наш T под специальные "требования"

Первый вариант не совсем очевидный, плюс, не подходит под современные стандарты STL. Поэтому, рассмотрим второ вариант:

template<typename T>
T do_nth_power(T x, T result, unsigned n) {
    while (n > 0) {
        if ((n & 0x1) == 0x1) { // проверка n на четность 
            result *= x;
            n -= 1;
        } else {
            x *= x;
            n /= 2;
        }
    }
    return result;
}

unsigned nth_power(unsigned x, unsigned n) {
    if (x < 2u || n == 1u)
        return x;
    return do_nth_power<unsigned>(x, 1u, n);
}

Также стоит не забыть про то, что мы не указали явно, что T - это копируемый тип (таким образом, чтобы его можно было перемножать). Для этого введем специальный concept:

template <typename T> concept multiplicative = requires(T t) {
    { t *= t } -> std::convertible_to<T>;
};

template<typename T>
T do_nth_power(T x, T result, unsigned n) 
requires multiplicative<T> && std::copyable<T>

Инстанцирование

Инстанцированием мы называем процесс порождения экземпляра специализации

template<typename T>
T max(T x, T y) {
    return x > y ? x : y;
}

...

max<int>(2, 3)

Специализация

Помимо инстанцирования, шаблонная функция или структура может быть явно специализирована:

template <typename T> T max(T x, T y) { ... }
template <> int max<int>(int x, int y) { ... }
template <> float max<float>(float x, float y) { ... }

Важные правила специализации:

  • Явное инстанцирование единожды в программе
  • Явная специализация единожды в программе
  • Явное инстанцирование должно следовать за явной специализацией
template<typename T> T max(T x, T y) { ... }
template <> int max<int>(int x, int y) { ... }
template int max<int>(int, int);

Также стоит учитывать, что явная специализация может войти в конфликт с инстанцированием:

template<typename T> T max(T x, T y);

// Указываем явную специализацию
template <> double max(double x, double y) {
    return 42.0;
}

// Никакой implicit instantiation
int foo() {
    return max<double>(2.0, 3.0);
}

// implicit instantionation нужен
int bar() {
    return max<int>(2, 3);
}

// ошибка: мы уже породили эту специализацию
template <> int max(int x, int y) {
    return 42;
}

Отсюда вытекает частный случай явной специализации - её запрет:

template <typename T> void foo(T*);

template<> void foo<char>(char*) = delete;
template<> void foo<void>(void*) = delete;

Non-type параметры

Non-type параметры - это структурные типы (т.е. скалярные типы, lvalue-ссылки и структуры, у которых все поля и базовые классы public и не mutable

struct Pair { int x, y; }

struct MoreComplex : public Pair {
    int arr[3]{ 1, 2, 3 };
    int z = 1;
}

template<int N, int* PN, int& RN, MoreComplex M> int buz() {
    return N + *PN + RN + M.z + M.arr[0];
}

Вывод типов

Для параметров, являющихся типами, работает вывод типов

int x = max(1, 2); // -> int max<int>(int, int);

Шаблонный тип аргумента может быть уточнен ссылкой или указателем и cv-квалификатором:

template<typename T> T max(const T& x, const T& y);
int a = max(2, 3); // -> int max<int>(const int&, const int&);

В некоторых случаях у нас нет контекста вывода, а поэтому мы можем указать наобходимое и положиться на вывод остального:

template<typename DstT, typename SrcT>
DstT implicit_cast(SrcT const & x) {
    return x;
}

double value = implicit_cast(-1); // <- ошибка
double value = implicit_cast<double, int>(-1); // <- ok
double value = implicit_cast<double>(-1); // <- ok

Допустим у нас есть параметр по умолчанию. Тогда, нам нужно будет указать его тип:

template<typename T> void foo(T x = 1.0);

foo(1); // <- ok
foo<int>(); // <- ok
foo(); // <- ошибка

Однако, мы можем это легко исправить:

template<typename T = double> void foo(T x = 1.0);

foo(1); // <- ok
foo<int>(); // <- ok
foo(); // <- ok

Перегрузка

На Лекции 1. Строки была поставлена задача написать перегрузку оператора ==. Вот один из вариантов такой перегрузки:

template<typename CharT, typename Traits, typename Alloc>
bool operator==(const basic_string<CharT, Traits, Alloc>& lhs, 
                const basic_string<CharT, Traits, Alloc>& rhs) {
    return lhs.compare(rhs) == 0;
}

Но у этого есть один очень серьезный недостаток: допустим, мы хотим справнить ("Hello" == str). В такое случае мы создаем лишнюю копию. Решением этой проблемы будет банальная перегрузка оператора ==:

// Ниже приведен вариант такой перегрузки в стандартной библиотеке C++

template<typename CharT, typename Traits, typename Alloc>
bool operator==(const basic_string<CharT, Traits, Alloc>& lhs, 
                const basic_string<CharT, Traits, Alloc>& rhs) {
    return lhs.compare(rhs) == 0;
}

template<typename CharT, typename Traits, typename Alloc>
bool operator==(const CharT* lhs, const basic_string<CharT, Traits, Alloc>& rhs) {
    return rhs.compare(lhs) == 0;
}

template<typename CharT, typename Traits, typename Alloc>
bool operator==(const basic_string<CharT, Traits, Alloc>& lhs, const CharT* rhs) {
    return lhs.compare(rhs) == 0;
}

Вот общие правила перегрузки в C++:

  1. Выбирается множество перегруженных имён
  2. Выбирается множество кандидатов
  3. Из множества кандидатов выбираются жинеспособные кандидаты
  4. Лучший из жинеспособных кандидатов выбирается на основании цепочек неявных преобразований для каждого параметра
  5. Если лучший кандидат существует и является единственным, то перегрузка разрешена успешно, иначе ill-formed

Также стоит учитывать, что поиск Кёнига в пространствах имён не сработает для шаблонных функций:

namespace N {
    struct A;
    template<typename T> int f(A*);
};

int g(N::A *a) {
    int i = f<int>(a); // <- ошибка, < здесь оператор меньше 
    return i;
}

Вот как можно это исправить:

namespace N {
    struct A;
    template<typename T> int f(A*);
};

template<typename T> void f(int); // <- можно добавить любой параметр

int g(N::A *a) {
    int i = f<int>(a); // <- ok 
    return i;
}

Шаблоны классов

Частичный порядок

Шаблон может выиграть перегрузку

// Пример 1 - более очевидный
void foo(double x);

template<typename T>
void foo(T x);

foo(1); // <- это вариант с шаблоном, а не double

// Пример 2 - более нетривиальный
template<typename T> 
void f(T);

template<typename T> 
void f(T*);

int ***a;
foo(a); // <- здесь выберется вариант с указателем, то есть, с параметром T*

Во втором примере логика немного другая: и специализация, и инстанцирование, и вывод типов работают одновременно.

Имена в шаблонах

Рассмотрим следующий код:

int y = 2; { int y = y; }
int x = 2; { int x[x]; }

В данном примере первая строка является UB, так как так называемся точка определения объекта с именем y начинается в внутреннем scope, а поэтому не понятно как мы можем дать y значение y.

Точка определения (PoD) - это граница, в которой завершено объявление. До PoD имя не считается введённым в область видимости.

template<typename T>
struct fwnode /* PoD */ {
    T data_;
    fwnode *next_;
};

template<typename T> 
class Stack {
    fwnode<T> *top_;
};

Таким образом, например, мы можем использовать имя нашего объекта (допустим структуры) внутри тела структуры.

Рассмотрим следующий код:

#include <iostream>

template<typename T> struct D {
    enum { max = 10 };
};

char buffer[D<void>::max];
extern void clear(char* buf);

int main(int argc, const char* argv[]) {
    std::cout << "Requesting to clean " << D<void>::max << std::endl;
    clear(buffer);
    return 0;
}

Данный код является корректным, однако опасным. Все дело в том, что тип void в данном случае не совсем уместен, так как void - неполный тип.

Частичная специализация

Рассмотрим следующий класс Stack:

template<typename T>
struct fwnode /* PoD */ {
    T data_;
    fwnode *next_;
};

template<typename T>
class Stack {
    fwnode<T> *top_;
};

Для удобства тут было бы проще хранить массив указателей:

// Опустим стурктуру fwnode

template<typename T>
class Stack<T*> {
    T** content_;
};

Заметим, что такое упрощение имен работает в частичных специализациях.

template<typename T>
class A {
    A* a1; // <- здесь A означает A<T>
};

template<typename T>
class A<T*> {
    A* a2; // <- здесь A означает A<T*>
};

Для практики рассмотрим следующие примеры частичной специализации:

template<typename T, typename U> class Foo {  };            // Пусть это 1
template<typename T> class Foo<T, T> {  };                  // Пусть это 2
template<typename T> class Foo<T, int> {  };                // Пусть это 3
template<typename T, typename U> class Foo<T*, U*> {  };    // Пусть это 4

Foo<int, float> mif;        // <- это будет 1
Foo<float, float> mif;      // <- это будет 2
Foo<float, int> mif;        // <- это будет 3
Foo<int*, float*> mif;      // <- это будет 4

Стоит также учитывать очень важный момент: частично специализированный шаблон должен быть действительно менее общим, чем тот, версией которого он является:

template<typename T> class X {  };
template<typename U> class X<U> {  }; // <- это уже ошибка, так как ничем не отличается от верхней

Трюк Саттера

Для начала запомним главное правило: частичная специализация для функций невозможна.

template<typename T> void foo(T x);
template<> void foo<int>(int x); // <- здесь все нормально
template<typename T> void foo<int*>(T* x); // <- а вот тут уже ошибка

И отсюда возникает вопрос: есть ли вариант обойти такое ограничение? И на самом деле - да, оно есть. Это решение было предложено Гербом Саттером, называется такое решение Трюком Саттера. Суть заключается в делегировании реализации шаблонной функции к вспомогательной структуре, котороую можно специализировать по-разному в зависимости от типа:

#include <iostream>

template<typename T>
struct FImpl {
    static int f(T t) { 
        return 0; 
    }
};

// Частично специализируем структуру под указатели
template<typename T>
struct FImpl<T*> {
    static int f(T* t) {
        return 1;
    }
};

template<typename T> int f(T t) {
    return FImpl<T>::f(t); // <- теперь мы можем считать эту функцию "частично специализированный"
}

int main(int argc, const char* argv[]) {
    int firstValue = 12;
    std::cout << f(x) << std::endl; // <- здесь используется оригинал FImpl<T>
    
    int* firstPointer = &firstValue;
    std::cout << f(x) << std::endl; // <- а вот тут уже используется специализация FImpl<T*>
    return 0;
}

Также, стоит учитывать, что мы не можем назвать шаблонную специализацию наследованием, так как это нарушение одного из 5 принципов SOLID - LSP:

template<typename T>
struct S {
    void foo();
};

template<> struct S<int> {
    void bar();
};

S<double> sd;
sd.foo();

S<int> si;
si.bar();

Специализация может не иметь ничего общего с полной версией.

Параметризация методов и переходники

Рассмотрим следующую задачу:

#include <iostream>

template<typename T, typename U>
struct A {
    void func();
};

int main(int argc, const char* argv[]) {
    A<int, double> a;
    A<float, double> b;
    a.func(); // <- только для int
    b.func(); // <- для всего остального
    return 0;
}

Необходимо, чтобы метод был параметризован первым аргументом шаблона. Вот один из вариантов решения такой задачи:

#include <iostream>

template<typename T, typename U>
struct A {
    void func() {
        T dummy;
        some_func(dummy); // <- подбирается под тип
    }
private:
    // Общий вид функции
    template<typename V>
    void some_func(V) {
        std::cout << "For all" << std::endl;
    }
    
    // Специализация для типа int
    void some_func(int) {
        std::cout << "Only for int type" << std::endl; 
    }
};

int main(int argc, const char* argv[]) {
    A<int, double> a;
    A<float, double> b;
    
    a.func(); // <- работает только для int
    b.func(); // <- работает для всех
    return 0;
}

Но у такого решения есть минус: T dummy - это некоторый объект, размер которого не определен на стеке. Однако, мы можем исправить эту проблему достаточно просто - добавить так называемый переходник типов:

// Typ2Type - это переходник типов, который занимает 1 байт на стеке
template<typename T>
struct Type2Type {
    typedef T OriginalType;
};

template<typename T, typename U>
struct A {
    void func() {
        T dummy;
        some_func(Type2Type<T>()); // <- через Type2Type
    }
private:
    // Общий вид функции
    template<typename V>
    void some_func(V) {
        std::cout << "For all" << std::endl;
    }
    
    // Специализация для типа int
    void some_func(Type2Type<int>) {
        std::cout << "Only for int type" << std::endl; 
    }
};

int main(int argc, const char* argv[]) {
    A<int, double> a;
    A<float, double> b;
    
    a.func(); // <- работает только для int
    b.func(); // <- работает для всех
    return 0;
}
